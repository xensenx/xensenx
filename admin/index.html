How to Grant / Revoke Access

01
Generate a token

Run the following command to get a secure UUID. Use this as the map key in worker/whitelist.js. 

node -e "console.log('xns_' + crypto.randomUUID())"
02
Add to whitelist.js

Open worker/whitelist.js and add a new entry to the WHITELIST Map with the grantee's details.

03
Deploy the Worker

Push to GitHub (if using CI) or run wrangler deploy directly. The new token is live within seconds. 

wrangler deploy
04
Share the access link

Send the grantee a URL in the format below. They must visit it to set the auth cookie. After that, they can access /private directly. 

https://xensenx.com/private?token=xns_YOUR_TOKEN_HERE
05
Revoke access

Delete the token entry from whitelist.js and redeploy. The token becomes invalid instantly on the next Worker request. No session invalidation is needed — the cookie value simply won't match.
